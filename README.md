# Performance Review (MVP)

Локальное web-приложение для ежемесячной оценки сотрудников, квартальной агрегации результатов и распределения бонусного фонда по отделам.

## Что реализовано

- Отделы: создание, редактирование, деактивация.
- Сотрудники: создание, редактирование, фильтрация по отделу.
- Критерии оценки: создание/редактирование, веса, проверка суммы весов активных критериев.
- Месячная оценка:
  - ввод оценок 1..10 по критериям;
  - обязательный комментарий при оценке `<=4` или `>=9`;
  - расчет итогового месячного балла сотрудника;
  - расчет среднего месячного балла отдела;
  - завершение и блокировка периода.
- Квартальный обзор:
  - баллы за 3 месяца;
  - квартальные пункты (сумма месячных баллов);
  - квартальный средний балл сотрудника;
  - квартальный средний балл отдела.
- Премия:
  - ввод бонусного фонда;
  - порог участия (`avg_score` или `points`);
  - режимы распределения (`eligible_full_points`, `above_threshold_only`);
  - исключение неактивных сотрудников на дату расчета (MVP правило);
  - округление до 2 знаков и распределение остатка по наибольшей дробной части.
- Отчеты:
  - отчет по сотруднику;
  - отчет по отделу.
- Экспорт: CSV и XLSX квартального отчета.
- Backup SQLite базы через кнопку в UI.

## Технологии

- Python 3.9+
- Flask
- SQLite
- HTML/CSS/Vanilla JS

## Запуск

1. Перейдите в проект:

```bash
cd "/Users/katja/Documents/New project/Performance Review"
```

2. Создайте виртуальное окружение и установите зависимости:

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

3. Запустите приложение:

```bash
python3 backend/app.py
```

4. Откройте браузер:

[http://127.0.0.1:5000](http://127.0.0.1:5000)

## Структура

- `backend/app.py` — Flask API и маршруты.
- `backend/schema.sql` — схема SQLite.
- `backend/db.py` — инициализация БД и сид критериев.
- `backend/calculations.py` — расчетные функции.
- `backend/templates/index.html` — интерфейс.
- `backend/static/app.js` — фронтенд-логика.
- `backend/static/styles.css` — стили.

## Важно (MVP ограничения)

- Статус месяца хранится на уровне периода (год+месяц), без раздельного статуса по отделам.
- Для расчета премии требуется, чтобы все 3 месяца квартала имели статус `completed` или `locked`.
- История изменений полей и роли доступа не включены (отложено по ТЗ).
